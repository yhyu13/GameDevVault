## 🎯 核心原则

1. **双向链接优先**：Obsidian 的网状结构比文件夹更适合引擎知识。遇到相关概念时，用 `[[链接]]` 连接。
2. **以我为主**：不是论文搬运工，而是记录「**这对我当下的工作/项目有什么启发**」。
3. **输出倒逼输入**：每读一篇论文、看一段源码，必须有笔记输出。没有输出 = 没有发生。
4. **定期复盘**：每周五晚 30 分钟整理，每月一次深度复盘。
