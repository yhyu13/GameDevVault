# Agent 日志审查

> **执行时间**：每日工作结束前
> **预计耗时**：5-10 分钟
> **目的**：审查 Agent 当日执行日志，识别错误模式、性能瓶颈、用户反馈

---

## 审查维度

### 1. 错误与异常
- [ ] 统计当日工具调用失败次数与原因
- [ ] 识别高频错误模式（如 `old_string not found`、`timeout`、`API rate limit`）
- [ ] 记录到 `50-Knowledge-Management/Error-Pattern-Library.md`

### 2. 性能与效率
- [ ] 平均单次对话轮数
- [ ] 最长耗时任务及原因
- [ ] 工具调用成功率（成功/总调用）

### 3. 用户交互质量
- [ ] 用户是否重复询问同一问题（说明理解/回答不到位）
- [ ] 用户是否手动修正了 Agent 的输出（说明输出质量不足）
- [ ] 用户反馈关键词（"不对"、"错了"、"重新"、"改"）

### 4. 技能与工具使用
- [ ] 哪些技能被频繁调用？是否需要优化？
- [ ] 哪些工具从未使用？是否可以移除或推广？
- [ ] 新技能/工具 onboarding 效果如何？

---

## 审查模板

```markdown
## 日期：YYYY-MM-DD

### 错误统计
- 工具失败：X 次（原因：...）
- 逻辑错误：Y 次（场景：...）

### 新增错误模式
- [模式描述] → 解决方案/预防措施

### 性能观察
- 最长任务：XXX（耗时 XX min，原因：...）
- 工具超时：X 次

### 改进 Action
- [ ] 修复 ...
- [ ] 优化 ...
- [ ] 记录到知识库 ...
```

## 快速日志定位

```bash
# 搜索当日 ERROR 关键词（根据实际日志路径调整）
grep -i "error\|failed\|timeout\|exception" /path/to/logs

# 统计工具调用频率
grep -c "tool_call" /path/to/logs
```
