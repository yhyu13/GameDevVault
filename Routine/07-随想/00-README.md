# 07-随想 — 战略反思 / 树枝脉络类（GameDevVault）

## 战略反思的格式

不像 06 详细复盘有固定模板，**战略反思是自由形式**，但建议结构：

```markdown
---
tags: [随想, routine-reflection, strategic-thinking, 2026-07]
date: YYYY-MM-DD
aliases: [<关键词 1>, <关键词 2>]
---

# 战略反思：<一句话主轴>

> **写于 <触发事件> 之后。**
> 一段开场白：什么让我想写这篇？

## 一、先画树：什么是什么
## 二、问题诊断：养分在哪里错配了
## 三、如何把握三者的节奏
## 四、对当前 Routine 体系的结构性反思
## 五、从树到林（可选）
## 六、结语
```

## 写战略反思的触发条件

满足任一条件就写：

- **W 末**（每周日复盘完成后）：发现 Routine 体系结构性偏差
- **月末**：04 性能 / 06 复盘 / 07-周报 三个目录配比失衡
- **季度末**：主枝叶三层结构需要重画（如 O1/O2/O3 OKR 大调整）
- **承诺兑现率 V 字反弹**（如 W28 30% → W29 70%）：值得写一篇"为什么反弹 + 怎么继续涨"
- **跨主题嫁接点**（如 Lumen Surface Cache → Mac Metal RHI）：一篇可推动 2+ 林层
