# 技能审计 (Skill Audit)

> **执行时间**：每周一次
> **预计耗时**：20-30 分钟
> **目的**：审查所有 Skill 的质量、准确性、使用频率，确保技能体系健康

---

## 审计维度

### 1. 技能质量
- [ ] SKILL.md 格式是否规范？（YAML frontmatter + markdown body）
- [ ] 触发条件是否准确？是否覆盖预期场景？
- [ ] 步骤是否清晰、可执行？有无模糊描述？
- [ ] 是否有错误处理/异常路径？

### 2. 技能准确性
- [ ] 技能中的工具调用是否仍然有效？
- [ ] 引用的文件路径是否正确？
- [ ] 命令/代码是否可执行？
- [ ] 外部链接是否有效？

### 3. 使用频率
- [ ] 高频技能是否需要优化/简化？
- [ ] 低频技能是否需要推广或移除？
- [ ] 从未使用的技能是否还有保留价值？

### 4. 技能间关系
- [ ] 是否存在技能 A 和技能 B 解决同类问题？
- [ ] 技能之间是否可以组合产生新能力？
- [ ] 是否有技能依赖未记录的隐性知识？
