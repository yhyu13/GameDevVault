# Prompt 工程手册

> **目标**：收集和整理高质量 Prompt 模式，提升 Agent 的输出质量与可靠性
> **更新频率**：持续积累

---

## 核心原则

1. **具体 > 笼统**："写一个函数" → "写一个 Python 函数，接收 CSV 路径，返回 pandas DataFrame，包含错误处理"
2. **结构化 > 自由文本**：使用编号、列表、表格引导输出格式
3. **示例 > 描述**：提供输入/输出示例（Few-shot）比描述期望更精确
4. **边界 > 默认**：显式声明边界条件、异常处理、默认值
5. **分步 > 端到端**：复杂任务要求逐步思考（Chain-of-Thought）

---

## 场景模式

### 1. 代码生成

```markdown
任务：编写一个 [语言] 函数，实现 [功能]。

要求：
1. 输入：[参数定义]
2. 输出：[返回类型与格式]
3. 错误处理：[异常场景]
4. 风格：[命名规范、注释要求]
5. 示例：
   - 输入：...
   - 输出：...

约束：
- 不得使用 [禁用方法]
- 必须兼容 [版本/环境]
```

### 2. 调试诊断

```markdown
问题：[现象描述]

环境：
- 系统：...
- 版本：...
- 相关代码/配置：...

已尝试：
- ...

请按以下步骤诊断：
1. 列出可能的根因（3-5个）
2. 每个根因的验证方法
3. 推荐的修复方案
```

### 3. 知识检索

```markdown
问题：[具体问题]

要求：
- 优先使用 [指定数据源/工具]
- 如果无法验证，明确说明
- 提供引用来源
- 区分事实与推测
```

### 4. 任务委派（子智能体提示）

```markdown
任务：[明确的目标]

你已知信息：
- [背景上下文]

请执行：
1. [步骤1]
2. [步骤2]

输出要求：
- [格式]
- [必须包含的字段]
- [字数/范围限制]

注意：
- [边界条件]
- [常见陷阱]
```

## 输出格式控制

| 期望格式 | 提示关键词 |
|----------|-----------|
| Markdown 表格 | "以表格形式呈现" |
| 编号列表 | "按优先级列出 1. 2. 3." |
| 代码块 | "用 ``` 代码块包裹" |
| JSON | "返回 JSON 格式，字段名为..." |
| 结构化文本 | "使用 YAML frontmatter + markdown body" |

---

## 常见错误模式

| 错误提示 | 问题 | 修正 |
|----------|------|------|
| "请帮我..." | 过于笼统 | 明确输入、输出、约束 |
| "分析一下" | 无分析维度 | 指定分析框架/角度 |
| "尽可能好" | 无法衡量 | 定义明确的验收标准 |
| "不要..." | 负面约束易被忽略 | 用正面约束替代 |

## 高效 Prompt 模板库

### 代码审查
```markdown
请审查以下代码，按以下维度：
1. 正确性：逻辑是否正确？
2. 健壮性：错误处理是否完善？
3. 可读性：命名、注释是否清晰？
4. 性能：是否有优化空间？
5. 安全性：是否有潜在风险？

代码：
```

### 工具调用规划
```markdown
用户请求：...

可用工具：...

请规划：
1. 需要调用哪些工具？
2. 每个工具的输入参数？
3. 工具之间的依赖顺序？
4. 失败时的降级方案？
```
