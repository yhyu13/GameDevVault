# Agent 上下文窗口优化

> **目标**：在有限的上下文窗口内，最大化信息传递效率，避免关键信息丢失
> **适用场景**：长对话、多轮工具调用、复杂任务拆解

---

## 核心策略

### 1. 信息分层
- **L1 关键**：必须保留的信息（用户目标、约束、错误状态）
- **L2 重要**：当前任务相关（中间结果、待办列表）
- **L3 背景**：辅助理解（环境信息、历史记录）
- **L4 可弃**：已完成的细节、已验证的假设

### 2. 主动压缩
- 定期总结已完成的部分："截至目前，已确认：1. ... 2. ..."
- 将详细结果替换为摘要："详细结果已保存至 X，摘要如下：..."
- 移除过时的待办和已完成的步骤

### 3. 引用 > 复制
- 使用文件路径引用，而非复制文件内容
- 使用 `Read` 按需读取，而非一次性加载
- 使用技能名引用，而非展开技能内容

### 4. 结构化 > 自然语言
- 使用列表、表格、YAML 代替段落
- 使用标准模板减少重复描述
- 使用缩写和约定（如 P0/P1/P2）

---

## 优化技术

### 技术1：渐进式披露
只提供当前步骤所需的上下文，后续步骤按需补充。

### 技术2：状态机表示
用简短的状态标识代替长描述：
```
状态：等待用户确认 | 工具执行中 | 等待 API 响应 | 已完成
```

### 技术3：分块引用
大文件分块处理，只传递当前块 + 文件元数据：
```
文件：X.md（共 1000 行，当前处理 1-100 行）
关键内容：...
```

### 技术4：摘要链
每轮对话保留上一轮摘要，而非完整历史：
```
[摘要] 已讨论：A、B、C。共识：...。待解决：D。
```

---

## 监控指标

- 平均每次对话轮数
- 触发上下文压缩的频率
- 压缩后用户是否重复信息（说明压缩过度）
- 工具调用失败率（是否因上下文截断导致参数错误）

---

## 常见陷阱

| 陷阱 | 现象 | 解决方案 |
|------|------|----------|
| 过度压缩 | 用户重复已说过的信息 | 提高关键信息层级 |
| 压缩不足 | 超出上下文限制，响应截断 | 更激进的摘要和引用 |
| 错误压缩 | 丢失否定词、条件词 | 使用结构化格式，避免自然语言摘要 |
| 忽视元数据 | 不知道文件已更新 | 保留时间戳和版本信息 |
