# Agent 调试协议

> **目标**：建立系统化的问题诊断流程，快速定位 Agent 行为异常的原因
> **触发条件**：Agent 输出错误、工具调用失败、行为不符合预期

---

## 调试流程

### 1. 问题收集
- [ ] 记录问题现象（用户输入、Agent 输出、实际结果、期望结果）
- [ ] 记录时间戳和环境信息（Workspace、工具状态、上下文长度）
- [ ] 收集相关日志（工具调用记录、错误堆栈）

### 2. 分类诊断

#### A. 工具层问题
- 工具调用失败？
- 参数格式错误？
- 工具返回数据异常？
- **处理**：检查工具文档、降级到替代工具、修复参数

#### B. 技能层问题
- 技能未被触发？（触发条件不匹配）
- 技能步骤错误？（old_string 不匹配、路径错误）
- 技能逻辑缺陷？（遗漏边界条件、顺序错误）
- **处理**：更新技能文档、修复步骤、添加验证

#### C. 上下文层问题
- 上下文过长导致信息丢失？
- 历史对话压缩导致关键信息丢失？
- 用户意图理解偏差？
- **处理**：优化提示、精简上下文、主动澄清

#### D. 知识层问题
- Vault Memory 信息错误/过时？
- 技能中的知识引用失效？
- 知识边界模糊？（两个技能都认为自己该处理）
- **处理**：更新记忆、修复引用、明确边界

### 3. 修复与验证
- [ ] 实施修复
- [ ] 在相同场景下验证修复是否有效
- [ ] 检查修复是否引入新问题（回归测试）
- [ ] 记录到 `50-Knowledge-Management/Error-Pattern-Library.md`

### 4. 预防
- [ ] 是否需要更新技能模板以防止类似问题？
- [ ] 是否需要新增监控/检查点？
- [ ] 是否需要更新健康检查脚本？
