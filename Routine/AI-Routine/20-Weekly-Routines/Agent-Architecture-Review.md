# Agent 架构审查

> **执行时间**：每周一次（建议周一上午或周五下午）
> **预计耗时**：30-60 分钟
> **目的**：回顾 Agent 的整体架构设计，确保技能、工具、记忆、路由的协调一致

---

## 审查维度

### 1. 技能体系 (Skills)
- [ ] 技能数量是否过多？是否有冗余/重叠？
- [ ] 技能命名是否一致？是否遵循 `lowercase-hyphens`？
- [ ] 技能更新频率：哪些技能经常被更新？是否稳定？
- [ ] 技能之间的依赖关系是否清晰？

### 2. 工具集成 (Tools)
- [ ] MCP 工具与原生工具的分工是否合理？
- [ ] 工具调用链是否过长？是否可以组合优化？
- [ ] 新工具 onboarding 流程是否完善？

### 3. 记忆系统 (Memory)
- [ ] Vault Memory 是否及时更新？
- [ ] 会话记忆压缩是否正常？是否有重要信息丢失？
- [ ] 技能中的知识是否与 Vault Memory 重复？

### 4. 子智能体编排 (Subagents)
- [ ] `Agent` 工具的使用是否合理？有无过度使用？
- [ ] 子智能体的 context handoff 是否高效？
- [ ] 子智能体超时设置是否恰当？

## 架构原则 Review

1. **Single Responsibility**：每个技能只负责一个清晰的任务类型
2. **Fail Fast**：工具不可用时应立即降级，而非隐藏错误
3. **Context Economy**：减少不必要的 context 传递，避免 token 浪费
4. **Observability**：所有重要操作都应可记录、可审查
