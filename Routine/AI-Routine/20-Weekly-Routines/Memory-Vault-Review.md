# 记忆 Vault 审查

> **执行时间**：每周一次
> **预计耗时**：15-20 分钟
> **目的**：确保 Vault Memory 的内容准确、有用、不过时，且与技能体系互补

---

## 审查维度

### 1. Vault 内容质量
- [ ] `about_user.md` 是否准确反映用户当前偏好？
- [ ] 是否有重复或矛盾的信息？
- [ ] 信息是否过于笼统？（如 "用户喜欢编程" → 应具体到领域/技术栈）
- [ ] 是否有敏感信息泄露风险？

### 2. 记忆时效性
- [ ] 是否有已过时但仍保留的信息？
- [ ] 用户的偏好/目标是否已变化但未更新？
- [ ] 是否有临时信息被长期保存？

### 3. 记忆与技能的边界
- [ ] Vault 中是否有应放入技能的内容？（如流程、模板）
- [ ] 技能中是否有应放入 Vault 的内容？（如用户偏好、个人数据）
- [ ] 边界是否清晰：Vault = 用户事实；Skill = 可复用流程

### 4. 记忆压缩质量
- [ ] 会话压缩后是否保留了关键信息？
- [ ] 压缩后的内容是否仍然可理解？
- [ ] 是否有重要信息在压缩中丢失？

## 记忆健康度指标

- **准确率**：信息是否与用户实际一致？（自评 A/B/C）
- **时效性**：是否有超过 3 个月未更新的关键信息？
- **冗余度**：同一信息在多少处重复？
- **覆盖率**：用户画像是否完整？（职业、偏好、目标、限制）
