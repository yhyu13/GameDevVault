## 周报 4 段式硬约束

每份周报必须包含 4 段（来自 README "输入 → 输出 → 连接" 闭环）：

1. **输入（Input）— 本周读了什么 / 学了什么**
   - 论文、GDC 演讲、源码、技术雷达条目
   - 引用 [[01-论文笔记库]] / [[02-引擎源码分析库]] / [[05-技术雷达]] 等具体笔记

2. **输出（Output）— 本周产出了什么**
   - 写了几篇笔记 / 案例 / 源码分析
   - GitHub commits 列表（带 hash 链接）
   - QA 卡 / 工具脚本

3. **连接（Connection）— 本周对外建立了什么连接**
   - 内部 cross-link（如 03-shader 配 04-性能 的案例）
   - 外部（博客 / 技术沙龙 / 同事交流 / 提的 PR）

4. **承诺兑现率**（周末必填）
   - 周一开盘定了几个目标 → 周日复盘兑现几个
   - 失信项 + 失信原因 + 下周补救计划
