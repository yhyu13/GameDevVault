## 一、三线程模型 [D]

```
GameThread (GT)  →  Tick 逻辑、Actor、动画、物理、蓝图
      │ ENQUEUE_RENDER_COMMAND
      ▼
RenderThread (RT)  →  生成平台无关的 RHI Command List
      │ ImmediateFlush (DispatchToRHIThread)
      ▼
RHIThread (可选)  →  翻译成 DX12 / Vulkan / Metal 调用
      │ 提交 GPU
      ▼
GPU
```

- **GT 跑 N 帧，RT 跑 N 或 N-1 帧，RHI 跑 N 或 N-1 帧**——允许落后一帧，最大并行（来源：UE 源码 + 社区博客）
- **GT 不能领先 RT 超过 1 帧**，否则 `FrameEndSync` 阻塞 GT（来源：同上）
- **RT 每一帧都 `ImmediateFlush` 等 RHI**（deferred shading 管线）→ RT 等 RHI = 卡（来源：同上）

## 二、各线程时间从哪看 [D]

`stat unit` 输出列（详见 `[[Unreal Insights 帧分析实战]]`）：

| 列 | 对应线程 |
|----|---------|
| Frame | 整帧（含 Present） |
| Game | GameThread |
| Draw | 渲染线程准备（部分场景） |
| GPU | GPU |
| RHI | RHI（提交到驱动的总开销） |
| RHIT | RHIThread（PC / 主机） |
| Present | 缓冲交换 + VSync |

→ **找瓶颈前先 `stat unit` 看各列比例**（官方推荐做法）。

## 三、`stat unit` 关键诊断结论（官方 / 经验共识）[D]

> 来源：[Unreal Insights Overview](https://docs.unrealengine.com/5.7/en-US/unreal-insights-overview-in-unreal-engine) + [Frame Pacing](https://docs.unrealengine.com/5.7/en-US/frame-pacing-in-unreal-engine)

| 现象 | 看哪一列 |
|------|---------|
| 帧率高但 CPU/GPU 时间都不高 | Present 列（VSync 等显示器） |
| 单帧高，前后帧正常 | Game / Draw 列（GC / 同步加载 / Shader 编译） |
| 持续高帧 | Game / GPU 哪一列大 |
| GT 时间短，但帧时间长 | GT 在等 RT（FrameEndSync） |
| RT 时间高 | `profilegpu` 看 GPU Pass 排序 |
| GPU 时间高 | `profilegpu` 看 Pass 排序，看具体是 BasePass / Lighting / Lumen / Nanite |

## 四、相关 CVar（UE 源码可见）[D]

> CVar 名以 UE 源码为准（`Source/Runtime/RenderCore/Private/RenderThread.cpp` / `Source/Runtime/Engine/Private/PhysicsEngine/`）

| CVar | 作用 | 备注 |
|------|------|------|
| `r.OneFrameThreadLag` | 允许 RT 落后 GT 一帧 | 默认 1 |
| `r.RHICmdBypass` | 关闭 RHI 线程（0 = 开 RHI 线程） | 默认 0 |
| `r.RHIThread` | 显式开关 RHI 线程 | 默认 0（移动）/ 1（PC / 主机推荐开） |
| `r.RayTracing.Culling.Radius` | 远场追踪剔除半径 | 与 Lumen Far Field 配 |
| `p.AsyncPhysicsTickEnabled` | 物理异步 tick | UE5 默认开 |
| `p.MaxPhysicsSubsteps` | 物理最大子步数 | UE5 默认值 |

## 五、移动端特殊 [D]

> **来源**：UE 官方 [Mobile Rendering](https://docs.unrealengine.com/5.7/en-US/mobile-renderer-in-unreal-engine) 文档

- 移动 GPU 是 **TBDR**（Tile-Based Deferred Renderer），渲染路径不同
- 移动 CPU 是 **big.LITTLE**，线程亲和性重要
- **iOS 默认不开 RHI 线程**（来源：UE 源码 + 社区博客）
- **Android 视情况**：Adreno / Mali 各有偏好；移动端打开 RHI 线程前要 Profile
- 移动端 **MSAA 几乎不要用**（TBDR 本身 deferred）
- 永远检查 `r.Mobile.*` 系列 CVar

## 六、`stat` 命令清单（UE 官方 stat 系统）[D]

> **来源**：UE 官方 stat 命令文档

| 命令 | 显示内容 |
|------|---------|
| `stat unit` | 帧各线程总时间 |
| `stat unitgraph` | 帧时间曲线 |
| `stat game` | GameThread 各 TickGroup 时间 |
| `stat gpu` | GPU 总时间 |
| `stat memory` | 内存总量 + 增量 |
| `stat slow` | 慢 Actor 列表 |
| `stat actor` | Actor 列表 |
| `stat startfile / stat stopfile` | 启停 stat 文件录制 |
| `profilegpu` | 当前 GPU Pass 列表（一次性 dump） |
