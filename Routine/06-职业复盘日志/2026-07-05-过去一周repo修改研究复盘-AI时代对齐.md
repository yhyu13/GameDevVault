> 路径：`Career/Kimi/UE5_Extending_MCP/`
> **本质**：把 Epic 官方 `Engine/Plugins/Experimental/ModelContextProtocol` 从"15 个玩具工具"扩成"15 个生产可用工具 + 完整的工程化流水线"。

| 9 | 最终集成 + 3 类关键 fix | `b34b492` | ✅ 11 PASS / 3 GATED |

**Phase 9 三大真实工程坑**（来自 `phase9_final_integration/README.md`）：
1. **Game thread deadlock** — `snapshot_world` / `restore_world` / `list_blueprints` 因 `LoadPackage → FlushAsyncLoading` 在 HTTP server 线程自死锁。修复：`IFileManager::Copy` 替代 `LoadPackage`，加 `IsAssetLoaded()` 守卫。
2. **JSON 格式 bug** — `FModelContextProtocolToolResult` 直接传 `TSharedPtr<FJsonObject>` 而没包 `content: [{type:"text"}]`，HTTP 响应 body 为空。修复：所有工具统一 `FJsonSerializer::Serialize → MakeTextResult`。
3. **Class lookup 三级 fallback** — `FindObject<UClass>` 找不到 native class（`StaticMeshActor`）也找不到 BP（缺 `_C` 后缀）。修复：`FindFirstObject<UClass>(ExactClass)` → 加 `_C` → `TObjectIterator<UClass>` 兜底。

**Phase 9 之后 Run 9 smoke test 又炸出一个新坑**（来自 `tests/smoke_test_results_20260701.md`）：
- `set_property` 工具用 `FindFieldChecked`（hard assert）在 Brush_1 上找 `bHiddenInGame` → 找不到 → `assert` → **编辑器 Fatal error 崩溃**。
- 根因：`FindFieldChecked` 只查 leaf class 不查继承成员，而 `bHiddenInGame` 在父类 `AActor`。
- 教训：**任何允许 LLM 输入的工具都必须用 `FindField` + nullable fallback，不能用 `Check` 系**。这是 MCP 工具开发的一个通用 pattern。

**对齐价值**：这条主线 = **MCP = Agent TCP/IP**（参见 [[05-技术雷达]] 记忆中的 10 大跨 vault 模式第 1 条）的真实落地。不是写"我们支持 MCP"，是写出"我们通过 MCP 让 15 个工具 PASS、3 个 GATED（需要显式启 mutation）、并捕到一次真实的 Fatal error"。
