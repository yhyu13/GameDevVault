## 写作风格 (2026-07-01 锁定)

每篇 shader 案例必须 **双轨交付**：

1. **可跑代码** — 完整 HLSL/GLSL，能直接拷进项目，配合参数面板就能跑出效果
2. **概念拆解** — 为什么这样写、为什么这个参数、为什么这样降级，把"手感"也写下来

**只给代码不给推导 = 没灵魂；只给推导不给代码 = 用不上。**

## 收集策略

Shader 是需要手感的技能。本库记录：

1. **自己写的** — 带完整代码和参数解释
2. **网上看到的优秀案例** — 注明来源，分析为什么好
3. **工作中复用的** — 可快速拷贝的片段

## 文件夹结构

- **[[后处理效果]]** — SSR、体积云、Bloom、TAA、色调映射
- **[[材质系统]]** — PBR、各向异性、Subsurface、Clear Coat
- **[[计算着色器]]** — Compute Shader 通用计算
- **[[顶点/几何效果]]** — 顶点动画、曲面细分、Geometry Shader

---

## 案例记录标准

每个案例必须包含：

1. **效果截图** — 正面/反面教材都要有
2. **核心 HLSL/GLSL 代码** — 注释清晰
3. **参数解释** — 每个 uniform 的含义和推荐范围
4. **性能分级** — 移动端能用吗？需要降采样吗？
5. **变体版本** — 低配/高配两种实现
