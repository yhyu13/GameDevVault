## 为什么从 [[Meshy-LumaGenie-Text-to-3D]] 拆出来独立

在 2026-06-26 的 P1 条目里,Hunyuan3D 只是 Meshy / Luma / Tripo3D 三家对比表里的一项。但 2026-07-03 复评时识别出 **三个独立价值**:

1. **拓扑质量:优(带拓扑)** vs Meshy/Luma "中(可后处理)"
2. **开源可商用** vs Meshy/Luma 闭源按量计费
3. **直接挂骨骼动画** vs 其他家要 retopo 后再绑骨

这三件事叠加,**让 Hunyuan3D 从"对比表里的一项"变成"主力选择"** — 当你需要做"AI 生成的 NPC 角色接 UE Skeletal Mesh"时,Hunyuan3D 是当前唯一靠谱的端到端方案。

---

## 关键能力(2026 H1)

- **Image-to-3D**:输入单张图,出带拓扑的 mesh(30 秒-2 分钟)
- **专业拓扑布线**:符合动画绑定 / UV 展开的 mesh 结构
- **多格式输出**:FBX / OBJ / GLB / USDZ,主流引擎直接接
- **开源可商用**:腾讯开源协议,商业项目免授权费
- **风格一致性**:通过 LoRA / 参考图锁定画风(类似 2D 那套)

## 与当前工作的关联

- [x] 直接相关 — 当前项目可用(资产生成)
- [x] 间接相关 — 未来项目可能用(程序化内容生成 PCG)
- [x] 知识拓展 — 拓宽技术视野

**具体关联点:**

- **快速 prototype**:策划说"我要一堆史莱姆",AI 一下午出 50 个变体
- **NPC 角色变体**:Hunyuan3D 的"专业布线"让 AI 角色**直接挂 UE 的 Skeletal Mesh + Animation Blueprint**,跳过 retopo + 绑骨步骤
- **UGC 工具**:UE 项目里加"AI 生成你的角色"功能,Hunyuan3D API + WebSocket 推流
- **配合 [[UnrealMCP-N1UnrealMCP]]**:用 MCP 让 AI Agent 自动跑 Hunyuan3D + 导入 UE + 验证拓扑,完全闭环
- **程序化关卡**:AI 生成 + PCG 框架结合 — 美术给 prompt,引擎批量出场景元素

**对你 day-job 的真实杠杆:**

- **Meshy/Luma 闭源按量**:每次生成都要钱,迭代成本高
- **Hunyuan3D 开源可商用**:API 调通后基本零边际成本,可以放心大量生成
- **拓扑结构省 60% 后处理时间**:其他家 AI 出的 mesh 都要 retopo + UV 重展,Hunyuan3D 基本不用
