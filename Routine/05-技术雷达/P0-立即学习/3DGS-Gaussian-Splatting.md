**关键技术节点(2026 现状):**
- **原始 3DGS**(Kerbl 2023, SIGGRAPH):高斯椭球 + 可微光栅化,1080p 30fps
- **动态 3DGS / 4DGS**:支持动画角色 + 场景联动,2025-2026 是研究热点
- **物理仿真 + 3DGS**:把高斯当作粒子做物理(ArtisanGS、AHA!),开始能跑布料/碰撞
- **SLAM + 3DGS**:实时定位 + 重建,FeatureSLAM 等 2026 实时方案
- **Luma UE5 插件**:Windows 实时,1 分钟建模、$1 成本(对比传统 2 周 $1000)
- **腾讯 Hunyuan3D**:Image-to-3D 自带"专业布线结构",**直接进 UE 资产管线**

**对游戏开发的实际意义:**
- **真·扫描资产**:用手机/相机拍一圈就能生成可走动的关卡,质量在"远处"够用
- **真·动态光照**:神经表示天然带全局光照暗示,不需要再走烘焙
- **真·快速迭代**:场景美术不用从 0 搭场景,从实拍 → 3DGS → 引擎关卡

**关键技能点(数学 + 系统都重要):**
- **多视图几何**:相机内外参、极几何、束调整(BA)
- **可微渲染**:PyTorch 风格的自动微分怎么在光栅化管线里跑
- **GPU 优化**:radix sort 排序、tile-based 光栅化、内存带宽
- **资产压缩**:3DGS 一个场景动辄几 GB,游戏需要流式加载和 LOD

## 与当前工作的关联

- [x] 直接相关 — 当前项目可用(资产管线)
- [x] 间接相关 — 未来项目可能用(关卡生成、Lumen 替代)
- [x] 知识拓展 — 拓宽技术视野(从"网格+贴图"扩展到"神经表示")

**具体关联点:**
- **Unreal Engine 5.7+ NNE**:NNE 已经能加载 `.plan` / `.onnx` 格式的模型,可以挂自定义 3DGS 推理
- **Nanite vs 3DGS**:Nanite 是微多边形,3DGS 是神经表示 — **两者不是替代关系,是互补**:
  - 主角 + 关键道具:Nanite(精度可控、动画兼容好)
  - 远景 + 程序化场景 + 扫描资产:3DGS(开发快、视觉真)
- **Lyra 关卡生成**:用 3DGS 把真实照片直接转成可玩关卡,跳过传统建模
- **Hunyuan3D 的拓扑结构**:Image-to-3D 还直接出"能挂动画的 mesh",不是死的 splat — **这是 3DGS 不能完全替代 mesh 的关键证据**

**对你 day-job 的真实杠杆:**
- **快速搭原型**:关卡美术资源不够时,3DGS 30 分钟出可玩场景
- **Hunyuan3D 资产管线**:微信扫描 → 拓扑 mesh → UE 导入 → 加动画 → 资产就位,**这条管线已经打通,值得试一次**

## 我的判断(只对你说)

**3DGS 是"在 Lumen 之外,游戏引擎的第二个 AI-native 渲染入口"** — 一个是像素端的 AI(超分、降噪、帧生成),一个是表示端的 AI(场景本身就是神经的)。  
短期最大杠杆:**用 Luma UE 插件跑通一遍,体验"拍 30 张照片 → 引擎里能走"**,这个体验会彻底改变你对关卡美术工作流的理解。  
警惕陷阱:3DGS 的动态场景 / 物理 / 流式加载都不成熟,**别想用 3DGS 替代所有 mesh** — Nanite 和 3DGS 是**分工**关系,不是替代。
