# 每周外部接触：AI 任务清单 — Lumen 技术社交

> **人类目标**：打破信息茧房，验证技术路线，寻找合作/内推机会。  
> **AI 任务**：润色措辞、生成问题、准备资料，绝不替代社交互动。

## 任务 1：技术问题润色（AI 执行）

**输入**：你 raw 的技术问题，如 "lumen 的radiance cache 怎么更新啊，搞不明白"

**AI 输出**：润色后的版本（选择适合的平台）：

### 知乎/论坛版（更详细）
> 标题：UE5 Lumen 的 Radiance Cache 更新机制是怎样的？
> 正文：我正在阅读 Lumen 的 SIGGRAPH 2021 论文，对 Radiance Cache 的更新频率有些困惑。论文中提到 Radiance Cache 是动态更新的，但没有详细说明更新触发条件。我有以下几个具体问题：
> 1. Radiance Cache 是每帧全量更新，还是增量更新？
> 2. 更新频率与 Surface Cache 的更新频率是否一致？
> 3. 在光源移动时，Radiance Cache 的响应延迟是多少帧？
> 感谢指点！

### Twitter/X 版（更简洁）
> 读 Lumen paper 卡住了：Radiance Cache 是每帧全量更新还是增量更新？有没有大佬知道 UE5 源码里在哪看？#UE5 #Lumen #GraphicsProgramming

### Discord 群版（更 casual）
> 大佬们，Lumen 的 Radiance Cache 更新频率有了解的吗？我在看源码，但不太确定是每帧全量更新还是按区域增量更新。求指点 🙏

**你必须做**：确认润色后的版本是否准确表达了你的真实问题。如果 AI 添加了你不知道的技术细节，删除它们。

## 任务 4：社区贡献准备（AI 执行）

**输入**："我想给开源渲染器（如 Godot）或 Lumen 相关项目提一个 PR，但不确定做什么"

**AI 建议流程**：
1. **找 good-first-issue**：在 GitHub 上搜索标签 `good first issue` + `rendering` + `global illumination`
2. **评估匹配度**：AI 帮你评估哪个 issue 适合你的技能水平
3. **修复方案**：AI 和你一起讨论修复思路（不直接给代码）
4. **PR 描述**：AI 帮你写 PR 描述模板

**示例输出**：
> 找到 Godot 渲染器的一个 issue：#XXXX "Lumen-like GI 在 Mobile 上的 fallback 缺失"
> 你的技能匹配度：★★★★☆（你有自研引擎经验，了解 Mobile 限制）
> 建议：从添加一个简单的 Bake 按钮开始，调用现有 Lightmap 系统作为 fallback

**你必须做**：
1. 在 GitHub 上找到真实的 issue
2. 阅读 issue 历史和评论
3. 自己设计修复方案
4. 写代码、测试、提 PR
5. 维护社区关系（回复 review 评论）
