# 周三：AI 任务清单 — Unreal MCP 工具观察与体验

> **人类目标**：以开发者视角观察 AI 助手与工具交互的模式，体验现有 MCP 实现。  
> **AI 任务**：帮你结构化观察、生成对比表格，绝不替你做体验。

---

## 任务 1：观察清单生成（AI 执行）

**输入**："我要观察 AI 助手（如 Claude、Cursor）通过 MCP 与开发工具交互的模式。生成结构化观察清单"

**AI 输出**：

### MCP 交互模式观察清单

| 观察维度 | 具体观察点 | 记录方式 |
|----------|-----------|---------|
| **工具发现** | AI 如何学习可用工具？（Schema 解析 vs 硬编码） | 截图/日志 |
| **调用粒度** | 一次用户请求触发几次 Tool Call？ | 计数 |
| **错误恢复** | Tool 调用失败时，AI 如何重试或降级？ | 录屏 |
| **Context 管理** | 长对话后，AI 是否还记得之前的工具结果？ | 测试 |
| **权限边界** | AI 是否能执行危险操作？如何被限制？ | 实验 |
| **延迟感知** | Tool 调用延迟是否影响 AI 的回复流畅度？ | 计时 |

**推荐体验平台**：
- Claude Desktop + MCP 官方示例（Filesystem、GitHub、SQLite）
- Cursor 的 `@` 工具调用
- 本地搭建的 Unreal MCP（如果 UE58 已编译）
