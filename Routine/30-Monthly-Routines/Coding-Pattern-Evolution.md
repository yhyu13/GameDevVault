# 编码模式演进

> **执行时间**：每月一次
> **预计耗时**：30-45 分钟
> **目的**：回顾 Agent 的编码习惯与模式，提炼最佳实践，淘汰反模式

---

## 回顾维度

### 1. 代码风格一致性
- [ ] 是否遵循统一的命名规范？
- [ ] 错误处理是否一致（try-catch / error returns）？
- [ ] 日志/注释风格是否统一？

### 2. 架构模式
- [ ] 最常用的代码组织模式是什么？
- [ ] 是否有重复造轮子？（同一功能以不同方式实现多次）
- [ ] 模块间的耦合度是否合理？

### 3. 反模式识别
- [ ] 是否存在 "复制粘贴编程"？
- [ ] 是否有过度工程化的代码？
- [ ] 是否有 "硬编码" 魔法值？
- [ ] 是否忽视了边界条件？

### 4. 工具使用习惯
- [ ] 是否善用 Python 标准库？
- [ ] 是否正确使用外部包（如 pandas, numpy）？
- [ ] 代码执行（PythonRun）的输出处理是否健壮？
