# 新技能开发流程

> **目标**：规范新技能的创建流程，确保质量、可维护性、可触发性
> **触发条件**：用户需要创建新技能，或发现现有技能需要拆分/重构

---

## 开发流程

### 1. 需求分析 (10-15 min)
- [ ] 明确技能要解决的问题类型
- [ ] 确定触发条件（关键词、场景、用户意图）
- [ ] 检查是否已有技能可覆盖（避免重复）
- [ ] 评估是否需要拆分为多个技能（SRP 原则）

### 2. 研究与设计 (20-30 min)
- [ ] 参考现有同类技能的 SKILL.md 格式
- [ ] 设计技能的输入、输出、边界条件
- [ ] 确定所需的工具链（原生工具 / MCP / 外部 API）
- [ ] 规划错误处理路径

### 3. 编写 SKILL.md (30-60 min)
- [ ] 编写 YAML frontmatter（name, description, triggers）
- [ ] 编写 markdown body（步骤、示例、注意事项）
- [ ] 确保步骤可执行、无歧义
- [ ] 添加 Pitfalls 章节（常见错误和规避方法）

### 4. 测试与验证 (15-30 min)
- [ ] 在模拟场景中测试技能触发
- [ ] 验证技能步骤中的每个工具调用
- [ ] 检查错误处理路径是否有效
- [ ] 确认技能是否与其他技能冲突

### 5. 部署与注册 (5 min)
- [ ] 将技能文件保存到正确的技能目录
- [ ] 更新技能索引（如需要）
- [ ] 记录到 `20-Weekly-Routines/Skill-Audit.md`

---

## 质量检查清单

- [ ] 技能名：小写，连字符分隔，不超过 64 字符
- [ ] 触发条件：具体、明确、可区分
- [ ] 步骤：编号、顺序、每个步骤原子化
- [ ] 工具引用：使用精确路径和参数
- [ ] 错误处理：有明确失败分支
- [ ] 可验证：有验收标准或测试用例
