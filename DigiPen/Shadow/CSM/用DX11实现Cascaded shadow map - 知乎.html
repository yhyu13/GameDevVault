从ShadowMap开始

CSM的实现建立在ShadowMap的基础上，ShadowMap的实现网上有很多，我这里就不再复读了，只稍微列一下大制实现流程：

  • 以光源为摄像机，渲染出场景的深度图，并作为RTT传入shader。
  • 将光源相机V，P矩阵的逆矩阵传入shader，反推出世界空间中的像素在光源空间中的深度A。
  • 从深度图中采样深度B，若深度A大于深度B则说明该点被遮挡，处在阴影中；反之则不在阴影中。

这样做就可以获得简单的实时阴影。网络上大多数教程就教到这个地方，但我依葫芦画瓢写完后总有一个疑问：平行光的光源相机该放在哪？该弄多大？

平行光的阴影理应覆盖整个场景（或整个主摄像机视口）。对于一个小的场景或者摄像机固定的场景你当然可以把它放在一个能拍到完整场景或整个摄像机视口的地方并固定不动。但对于大的场景，如果用一个单独的静态相机包裹整个场景那么阴影贴图分辨率的利用率就过于低，导致阴影质量极差。那么该怎么控制这个光源相机呢？

确定平行光光源相机

解决方案是让光源相机始终跟随主摄像机运动，使光源相机的正交视口始终将主摄像机的透视视口包在里面。这样就可以在让平行光阴影区域覆盖整个可视区域的同时，保证阴影贴图的分辨率达到最大利用率。这一步解决了普通ShadowMap遗留问题的同时，也是实现Cascaded Shadow Map必要的一步。

那么具体怎么算呢？原理是以下几步：

  • 根据主摄像机的各种视口参数，求出视口八个顶点在世界空间的位置。
  • 将八个顶点转换到光源摄像机的视口空间中，求出能包含这八个顶点的包围盒。这个包围盒即为光源相机的正交平截头体（平行光的光源相机是个正交相机，视口其实是个方的六面体）。
  • 将计算出的光源相机的正交平截头体转回世界空间，光源相机的位置为正交平截头体近平面的中心，朝向和光源相同，视口长宽远近等参数根据视口空间的数据也很容易求出。有了这些参数即可唯一确定光源相机。

我们来写代码吧，首先计算出主摄像机视口的八个顶点在世界空间中的位置。顶点位置可以用相似三角形求，但更优雅的做法是使用投影矩阵来逆推。世界空间中的视口乘以相机的View和Projection矩阵后会被转换到齐次空间中变为单位正六面体，那我们把单位正六面体的顶点乘上Projection和View矩阵的逆矩阵就可以算出八个顶点在世界空间中的位置了。

std::shared_ptr<ArcCamera::FrustumCorners> ArcCamera::GetFrustumCorners(float nearPlane, float farPlane) {
	std::shared_ptr<ArcCamera::FrustumCorners> result = std::make_shared<ArcCamera::FrustumCorners>();
	auto cameraTrans = GameObject()->getComponent<ArcTransform>();
	ArcCamera::FrustumCorners viewSpaceCorners;
	mat4 tempPro = CalculatePerspectiveMatrix(DegreeToRadians(m_fieldOfView), m_viewportAspectRatio, nearPlane, farPlane);
	mat4 mat = Inverse(View() * tempPro);
	float3 vecFrustum[8];
	vecFrustum[0] = float3(-1.0f, -1.0f, 0.0f); // xyz
	vecFrustum[1] = float3(1.0f, -1.0f, 0.0f); // Xyz
	vecFrustum[2] = float3(-1.0f, 1.0f, 0.0f); // xYz
	vecFrustum[3] = float3(1.0f, 1.0f, 0.0f); // XYz
	vecFrustum[4] = float3(-1.0f, -1.0f, 1.0f); // xyZ
	vecFrustum[5] = float3(1.0f, -1.0f, 1.0f); // XyZ
	vecFrustum[6] = float3(-1.0f, 1.0f, 1.0f); // xYZ
	vecFrustum[7] = float3(1.0f, 1.0f, 1.0f); // XYZ

	for (int i = 0; i < 8; i++) {
		vecFrustum[i] = TransformCoord(vecFrustum[i], mat);
		//PrintFloat3(vecFrustum[i]);
	}
	result->nearCorners[0] = vecFrustum[0];
	result->nearCorners[1] = vecFrustum[1];
	result->nearCorners[2] = vecFrustum[2];
	result->nearCorners[3] = vecFrustum[3];

	result->farCorners[0] = vecFrustum[4];
	result->farCorners[1] = vecFrustum[5];
	result->farCorners[2] = vecFrustum[6];
	result->farCorners[3] = vecFrustum[7];

	return result;
}

这里要注意的点是DX中表示齐次空间的六面体是个2*2*1的神秘东西，我在这里卡了好一会。上面函数中View矩阵和Projction矩阵的运算我的小引擎做了一些封装，大家自己算起来也不难，实在要看怎么算的可以在我的Github里看源码，在此就不多讲了。

Cascaded Shadow Map原理

CSM的原理其实不难，大致如下：

  • 把主摄像机的视口按距离截成几段，每一段分别设置一个独立的光源相机，按上文的方法计算出相机参数。
  • 每个光源相机分别渲染出深度图，将深度图打包作为RTT传入Shader。
  • Shader中选出最合适的深度图进行采样，利用采样出的深度数据，使用和ShadowMap相同的方式计算出阴影系数。

下面我一步步讲。

主摄像机平截头体截取

这一步做的就是把主摄像机的平截头体根据一定的间距切成几截。Unity的默认间距在Quality设置里，如图：

这个比例应该根据具体场景进行设置，我就没按Unity的设。。。

截取后，我们把一个平截头体变成了多个平截头体（这里是4个）。现在我们创建四个光源相机与平截头体一一对应，并按照上面在ShadowMap中计算光源相机位置的方法分别计算出四个相机的各种参数。相机我封装的比较厚重，在这里就不贴代码了，反正写起来也就是算几个矩阵循环一下的事。

光源相机渲染深度图并传入Shader

这里做的事和ShadowMap其实是类似的，但有个小问题在于，如果你传入的是4张单独的贴图你将无法达到预期效果，因为在shader里（至少sm 4.0里），你是无法用非常量从贴图数组中取值的。直接在if中采样不同贴图也不可行，虽然我不知道原因但是试了确实跑不出来，感觉又是GPU处离if方式带来的一些问题，有知道为什么的大佬麻烦告诉我下orz。

在DX11中的解决方案有两种，一种是使用一个叫Texture2DArray的数据结构，这个结构存储了多个贴图，且允许在Shader中使用非常量取出（群里大佬说可以，本人未测试）；另一种方法是将各个层次的贴图存在同一张贴图的SubResource中。这里我使用的是后一种方法，将多个光源相机渲染出的深度图放在同一张RTT的多层mipmap中。

首先创建所需的数据结构，包括：

  • 一张用来存实际数据的贴图 （ID3D11Texture2D）
  • 一个用于写入数据的View，这里只需要写入深度所以用存深度的View（ID3D11DepthStencilView）
  • 一个用于将数据传入shader的View（ID3D11ShaderResourceView）
  • 设置ViewPort为贴图的大小。

代码如下：

bool ArcRenderToTexture::InitializeAsDepthBuffer(ID3D11Device* d3dDevice, int TextureWidth, int TexureHeight , int mipLevel)
{
	//CreateTexture2D创建的是一个贴图
	//CreateDepthStencilView创建的是一个用于写入的视图，与之前创建的贴图绑定，当dx把东西写入这个视图时会自动写到绑定的贴图上
	//CreateShaderResourceView创建的是一个用于输出的视图，与之前创建的贴图绑定，用于把贴图的内容传入shader

	//第一,填充深度视图的2D纹理描述符结构体,并创建2D渲染纹理
	D3D11_TEXTURE2D_DESC depthBufferDesc;
	ZeroMemory(&depthBufferDesc, sizeof(depthBufferDesc));
	depthBufferDesc.Width = TextureWidth;
	depthBufferDesc.Height = TexureHeight;
	depthBufferDesc.MipLevels = mipLevel;
	depthBufferDesc.ArraySize = 1;
	depthBufferDesc.Format = DXGI_FORMAT_R24G8_TYPELESS; //24位是为了深度缓存，8位是为了模板缓存
	depthBufferDesc.SampleDesc.Count = 1;
	depthBufferDesc.SampleDesc.Quality = 0;
	depthBufferDesc.Usage = D3D11_USAGE_DEFAULT;
	depthBufferDesc.BindFlags = D3D11_BIND_DEPTH_STENCIL | D3D11_BIND_SHADER_RESOURCE;  //注意深度缓存(纹理)的绑定标志
	depthBufferDesc.CPUAccessFlags = 0;
	depthBufferDesc.MiscFlags = 0;
	HR(d3dDevice->CreateTexture2D(&depthBufferDesc, NULL, &mDepthStencilTexture));


	//第二,填充深度缓存视图描述符结构体,并创建深度缓存视图
	D3D11_DEPTH_STENCIL_VIEW_DESC depthStencilViewDesc;
	ZeroMemory(&depthStencilViewDesc, sizeof(depthStencilViewDesc));
	depthStencilViewDesc.Format = DXGI_FORMAT_D24_UNORM_S8_UINT;
	depthStencilViewDesc.ViewDimension = D3D11_DSV_DIMENSION_TEXTURE2D;
	depthStencilViewDesc.Texture2D.MipSlice = 0;
	HR(d3dDevice->CreateDepthStencilView(mDepthStencilTexture, &depthStencilViewDesc, &mDepthStencilView));
	

	//第三,填充着色器资源视图描述符,并进行创建着色器资源视图,注意这是用深度缓存(纹理)来创建的，而不是渲染目标缓存(纹理)创建的
	D3D11_SHADER_RESOURCE_VIEW_DESC shaderResourceViewDesc;
	shaderResourceViewDesc.Format = DXGI_FORMAT_R24_UNORM_X8_TYPELESS; //此时因为是仅仅进行深度写，而不是颜色写，所以此时Shader资源格式跟深度缓存是一样的
	shaderResourceViewDesc.ViewDimension = D3D11_SRV_DIMENSION_TEXTURE2D;
	shaderResourceViewDesc.Texture2D.MostDetailedMip = 0;
	shaderResourceViewDesc.Texture2D.MipLevels = depthBufferDesc.MipLevels;
	HR(d3dDevice->CreateShaderResourceView(mDepthStencilTexture, &shaderResourceViewDesc, &mShaderResourceView));

	//第四，设置视口的属性
	mViewPort.Width = (float)TextureWidth;
	mViewPort.Height = (float)TexureHeight;
	mViewPort.MinDepth = 0.0f;
	mViewPort.MaxDepth = 1.0f;
	mViewPort.TopLeftX = 0.0f;
	mViewPort.TopLeftY = 0.0f;

	return true;
}

创建好了之后，使用渲染最近也就是精度最高区域的光源相机将场景中的物体渲染到刚刚生成的ID3D11DepthStencilView中。渲染前别忘了重设视口以及清空渲染缓冲区等操作。

这样我们就得到了一张在第0层级已经写入了深度数据的RTT，下面我们来把剩下的三层深度数据写入到它剩下的三层mipmap里。

DX11用于写入的View，包括DepthStencilView和RenderTargetView不支持多层mipmap，所以为了将剩下三层渲染成RTT我们不得不再开三个新的Texture2D和三个新的DepthStencilView。生成方法跟生成第一张类似，只需要每次把分辨率降低到与对应mipmap相同即可（也就是每次降低到一半）。用剩下的三个光源相机渲染场景中的物体并分别写入对应的RTT，我们就得到了所需的全部深度图。

接下来往第一张图的mipmap中写入。mipmap在DX中被视为贴图的SubResource，我们使用ID3D11DeviceContext::CopySubresourceRegion这个函数来写入。代码如下：

bool ArcRenderToTexture::WriteToSubResource(ID3D11DeviceContext* deviceContext, int mipLevel) {
	ID3D11Resource *pSrcResource;
	mipDepthStencilViewVector[mipLevel - 1]->GetResource(&pSrcResource);

	ID3D11Resource *pDstResource;
	mShaderResourceView->GetResource(&pDstResource);

	deviceContext->CopySubresourceRegion(pDstResource, mipLevel, 0, 0, 0, pSrcResource, 0, NULL);
	return false;
}

在Shader中对合适的贴图层级进行采样

这一步的具体做法是，在Pixel Shader中将像素从世界空间分别转换到每个光源摄像机的光源空间，判断像素所处的分辨率最高的贴图。伪代码如下：

for(i in 1到4)
{
    uv=当前像素点的世界坐标，投影到第i个光源矩阵内。
    if(uv在0~1范围)
    {
        采样shadowmap算阴影。
        break;
    }
}

为了分别计算一个像素在每个光源相机投影空间的位置，我们得把四个相机的View * Projection的逆矩阵全部传入shader。

完整Shader代码如下，其中计算阴影的函数为CSMCalculation。

Texture2D gTex : register(t1);
Texture2D gShadowMap1 : register(t2);
SamplerState gSamLinear : register(s0);
SamplerState gSamLinearShadow : register(s1);

cbuffer ConstantBuffer : register(b0)
{
	matrix World;
	matrix View;
	matrix Projection;
};

cbuffer DirectionLight : register(b1) 
{
	float3 LightDir;
	float Intensity;
	float4 LightColor;
	float3 LightPosition;
	float Pad1;
	matrix LightMatrix[4];
};

struct VertexIn
{
    float3 Pos  : POSITION;
    float3 Normal : NORMAL;
    float3 Tangent : TEXCOORD0;
    float2 Tex : TEXCOORD1;
};

struct VertexOut
{
    float4 PosH  : SV_POSITION;
    float4 Color : COLOR;	
	float2 Tex : TEXCOORD2;
	float3 Normal : TEXCOORD3;
	float4 WorldPos : TEXCOORD4;
	float4 LightSpacePos : TEXCOORD5;
};

float3 Ambient = float3(0.1f, 0.1f, 0.1f);

float CSMCalculation(float4 worldPos, float bias){
	float shadow = 0.0;
	for (int i = 0; i < 4; ++i){
		float4 lightSpacePos = mul(worldPos, LightMatrix[i]);
		float3 projCoords = lightSpacePos.xyz / lightSpacePos.w;
		projCoords = float3(projCoords.x * 0.5f + 0.5f, projCoords.y * -0.5f + 0.5f, projCoords.z);
		
		if (projCoords.x > 0.01 & projCoords.x < 0.99 
		& projCoords.y > 0.01 & projCoords.y < 0.99 
		& projCoords.z > 0.0 & projCoords.z < 1.0
		){
			uint width = 1;
			uint height = 1;
			uint numberOfLevel = 1;
			float divide = 0.0;
			gShadowMap1.GetDimensions(i, width, height, numberOfLevel);
			float2 texelSize = float2(1.0f / width, 1.0f / height);
			float tempShadowSum = 0.0;
			for (int x = -1; x <= 1; ++x)
			{
				for (int y = -1; y <= 1; ++y)
				{
					float2 tempUV = projCoords.xy + float2(x, y) * texelSize;
					float pcfDepth = gShadowMap1.SampleLevel(gSamLinearShadow, tempUV, i).r;
					tempShadowSum += (projCoords.z - bias) > pcfDepth ? 0.9 : 0.0;/**/
				}
			}
			tempShadowSum /= 9.0;
			shadow += tempShadowSum;

			break;
		}
	}
	return shadow;
}

VertexOut VS(VertexIn vin)
{
    VertexOut vout;
	vout.PosH = mul(float4(vin.Pos, 1.0f), World);
	vout.WorldPos = vout.PosH;
	vout.LightSpacePos = mul(vout.PosH, LightMatrix[0]);
	vout.PosH = mul(vout.PosH, View);
	vout.PosH = mul(vout.PosH, Projection);
	vout.Normal = normalize(mul(vin.Normal, (float3x3)World));

    vout.Color = float4(vin.Tangent, 1.0f);
	vout.Tex = vin.Tex;

    return vout;
}

float4 PS(VertexOut pin) : SV_Target
{
	float bias = 0.00002f;
	float4 Albedo = gTex.Sample(gSamLinear, pin.Tex);
	float diffuseIntensity = saturate(dot(pin.Normal, normalize(-LightDir)));
	float3 diffuse = diffuseIntensity * LightColor.rgb * Intensity;
	float shadow = CSMCalculation(pin.WorldPos, bias);
	float3 result = (1.0 - shadow) * Albedo.rgb * (0.5f + diffuse) + Ambient;
	
    return float4(result, 1.0f);
}

效果优化

写到这里大致的CSM阴影效果已经出来了，但还有一些需要优化的地方。

1. PCF造成的边缘漏光

注意到我为了柔化阴影边缘，对阴影贴图做了个9倍超采样。在视锥体截取比例不合适的情况下（比如两层阴影分辨率差距过大），超采样会导致层级的边缘处出现这样的白色锯齿：

造成这样锯齿的原因是PCF在采样到贴图边缘时，若采样点+offset所在位置超出边界，则会采样到不正确的值，导致计算出的阴影数值出错。简单的解决方案是在判断像素是否落在某层uv内部时留下一定的边界，也就是把类似于：

if (projCoords.x > 0 & projCoords.x < 1 & projCoords.y > 1 & projCoords.y < 1)

改成：

if (projCoords.x > 0.01 & projCoords.x < 0.99 & projCoords.y > 0.01 & projCoords.y < 0.99)

2. 两层阴影交界处边缘过于明显

同样是上图，我们可以看到两层阴影交界处有个很明显的分界线（学名叫Cascade seams）。分界线的解决方案是在分界处将两层阴影blend一下。可以设置一个距离参数，若处在该距离参数的范围内则同时采样并混合两个层级上的阴影。这个blend我懒得写。。偷一张微软的图来表示下能达到的效果：

未优化阴影抖动时

可以看见平移、旋转时阴影抖爆，这个抖动是因为阴影相机每帧都在重新计算投影矩阵，导致生成的阴影贴图每帧都会有细微差异。当相机移动时，阴影边缘的像素不断进出阴影区域导致边缘闪烁。

在像素层面上，阴影闪烁的原因是，随着阴影相机的运动，阴影贴图相对于采样点的移动并不是只包含单纯的平移和旋转，还包含了一定的缩放。平移和旋转时采样点平滑的从一个像素移动到另一个像素，但缩放时，阴影会在多个像素之间快速跳跃，这就是边缘闪烁产生的原因之一。

那么，我们能做些什么防止缩放的出现呢？

  • 保证阴影贴图的宽高比和阴影相机视锥体宽高比一致。
  • 保证阴影贴图中每个像素在世界空间中对应的大小不变。

对于前一点：阴影贴图宽高比恒定为1：1，所以需要保证阴影相机视锥体宽高比也为1：1，也就是说视口为正方形。

对于后一点：只需保证阴影相机视口大小固定即可。

也就是说，我们需要保证阴影相机的视口是一个大小固定的正方形。这计算起来并不难，上文中更新的代码的做法是将主摄像机视锥体最长的一条对角线长度作为阴影相机视口长和宽（主摄像机参数不变时这是一个固定值）；从主摄像机视锥体包围盒的中心，根据光源方向推出阴影相机位置；根据主摄像机视锥体在z轴上的最大长度推出光源相机视锥体的Far-Near，即可得到需要的全部参数。

产生闪烁的另一个原因在于像素的对齐，这一点微软的dx文档里有讲到。

微软对像素对齐的解释​docs.microsoft.com

根据视锥体的长宽和阴影贴图的大小求出一个系数，保证相机的位移和视锥体的长宽都是这个系数的倍数，具体计算方式的伪代码大概如下：

 系数 = 视锥体长宽 / 阴影贴图分辨率
 距离 /= 系数
 距离 = floor（距离）
 距离 *= 系数

这样做相当于把阴影相机的视锥体在xy方向上离散化成了正方形的小格子，有效防止摄像机运动或是浮点数精度等带来的不规律微小变化造成的阴影偏移。

其实还有一种做法是先求出主摄像机视锥体的包围球，阴影相机再根据包围球求出视锥体。这样做也能满足视锥体固定大小正方形的条件，但这么做阴影相机的视锥体一般比我们之前做法的要大，造成精度浪费。我代码里其实也写了，也可以防止抖动，想看源码的可以自己去看，在ShadowCameraController.cpp里。
