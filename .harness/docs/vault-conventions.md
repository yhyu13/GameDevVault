# Vault Conventions

> 所有 rein 的共享笔记规范。任何改动先在这里改，再同步到受影响的 rein。

## 一条笔记的最小骨架

```markdown
---
tags: [<category>/<status>]
aliases: []
---

# <标题>

<正文>

## 关联
- [[相关笔记1]] — 为什么相关
- [[相关笔记2]] — 为什么相关
```

## Frontmatter

| 字段 | 必填 | 说明 |
|------|------|------|
| `tags` | 是 | 至少一个，格式 `<大类>/<状态>` |
| `aliases` | 否 | Obsidian 反向链接别名，列表 |

### 常用 tag 词表

| 大类 | 状态值 | 含义 |
|------|--------|------|
| `paper` | `待精读` / `精读中` / `signed` / `待复现` / `已复现` | 论文笔记生命周期 |
| `source` | `浅度浏览` / `深度分析` / `已应用到工作` | 源码分析深度 |
| `shader` | `练习` / `案例` / `生产可用` | Shader 案例成熟度 |
| `perf` | `案例` / `复现` / `已应用到项目` | 性能优化笔记 |
| `radar` | `P0` / `P1` / `P2` / `已掌握` / `放弃` | 技术雷达优先级 |
| `career` | `面试` / `复盘` / `交流` | 职业复盘类型 |
| `diary` | `周一` … `周日` | 日记对应星期 |
| `review` | `周复盘` / `月复盘` / `季度复盘` | 复盘周期 |

## 命名约定

- 文件名用 kebab-case 或中文，**不要混用**。同一目录内风格一致。
- 文件名应可独立搜索：`<年份>-<会议>-<标题>` 或 `<引擎>-<模块>-<主题>`。
- 论文笔记：`<第一作者>-<年份>-<关键词>.md`，例：`Mueller-2019-real-time-neural-radiance-cache.md`。
- 源码分析：`<引擎>-<模块>-<主题>.md`，例：`unreal-renderer-lumen-global-illumination.md`。

## 链接规则

- 正文第一次提到核心概念必须加 `[[]]` 链接。
- 同一篇笔记里同一个概念只链一次（除非语境不同）。
- 笔记末尾"关联"段必须至少 2 条 `[[]]` 链接，否则视为孤立。
- 反向链接：写笔记时如果引用了尚未存在的概念，先创建 stub 笔记，再回头补内容。

## MOC（Map of Content）维护

- `知识图谱-MOC.md` 是根索引，按领域分块。
- 每块的第一级链接是**领域聚合笔记**（如 `渲染管线`、`引擎架构`），不是单篇笔记。
- 单一聚合笔记饱和（>15 条子链接）时，拆分子 MOC。
- 新增笔记后，问自己：它属于 MOC 的哪个领域？是否需要新建聚合笔记？

## 不要做的事

- 不要把付费资料的完整 PDF / 截图入库。
- 不要在主仓修改 `Rendering/`、`Unreal/` 子模块里的代码。
- 不要给单条笔记加超过 7 个 tag——状态信号会被稀释。
- 不要把"是否复现"用 `[x]` 复选框写在正文里；用 `tags: [paper/待复现]`。
